# 07-09 Plan - Rebuild Checkout Page Into Shopify-Like Order Summary Plus Embedded Checkout

## Objective

Make the checkout page feel familiar to Shopify shoppers while keeping Stripe embedded Checkout and Worker-owned session creation intact.

## Scope Boundary

This is the checkout shell and layout pass. Payment fields remain inside Stripe embedded Checkout. Browser state remains non-authoritative.

## Tasks

1. Rebuild `/store/[slug]/checkout/` into the design spike's two-column desktop layout:
   - main checkout panel
   - order summary panel
2. Add mobile order summary collapse above the Stripe embedded Checkout area.
3. Render cart line item, subtotal, availability, and secure-payment copy in the summary.
4. Keep Worker offer/variant reads as the source for checkout eligibility.
5. Mount Stripe embedded Checkout only after Worker `StartCheckout` succeeds.
6. Recover on direct checkout page load without prior cart state by reading static item data and Worker offer state.
7. Add tests for layout states, mobile summary state, direct-load recovery, Worker reads, and mock checkout mode.

## Verification

- checkout page visually separates checkout form area from order summary.
- mobile order summary can expand/collapse.
- direct checkout page load shows the canonical item-option summary before payment starts.
- embedded Checkout still mounts from Worker-returned `clientSecret`.
- mock checkout mode renders the local mock panel.
- unavailable/API-error states remain non-transactional.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No custom payment form.
- No Stripe internals styling beyond supported embedded Checkout setup.
- No shipping form before Phase 9.
- No discount codes.
- No paid-order authority.
