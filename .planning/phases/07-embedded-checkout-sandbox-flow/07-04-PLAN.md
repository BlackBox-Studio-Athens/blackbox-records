# 07-04 Plan - Mount Embedded Stripe Checkout From Worker-Created Sessions

## Objective

Let the checkout shell start a Worker-owned Checkout Session and mount Stripe embedded Checkout with the returned client secret.

## Scope Boundary

This commit only starts checkout and mounts the embedded payment UI. It does not add webhook authority, order rows, stock decrement, shipping, or return/retry handling.

## Tasks

1. Add the approved Stripe.js dependency and frontend loader pattern.
2. Call `StartCheckout` with `storeItemSlug` and `variantId` only.
3. Mount embedded Checkout in the existing checkout shell mount zone using the Worker-returned `clientSecret`.
4. Keep Stripe price IDs, server secrets, and actor identity out of browser payloads.

## Verification

- checkout shell mounts embedded Checkout from Worker session data
- browser request payload does not include Stripe IDs or secrets
- unavailable items cannot start checkout
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass

## Explicit Exclusions

- No webhook handling
- No order persistence
- No stock decrement
- No shipping or BOX NOW flow
