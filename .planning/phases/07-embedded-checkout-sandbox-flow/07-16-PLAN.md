# 07-16 Plan - Validate The Local And Sandbox Checkout Loop With Real Stripe Sandbox Mappings

## Objective

Prove the Phase 7 checkout loop locally and in sandbox with selected real Stripe sandbox price mappings, after Stripe account access exists.

## Scope Boundary

This commit captures validation evidence and any required docs/scripts. Do not implement webhooks, order authority, stock decrement, shipping, or production cutover.

## Current Status

Stripe test account access exists, sandbox Worker secrets are configured, real sandbox `VariantStripeMapping` rows exist, and the deployed sandbox checkout loop has automated Playwright evidence.

Required local/sandbox setup:

- real Stripe test secret key (`sk_test_*`)
- real Stripe test Price ID (`price_*`) for selected mapped variants
- Stripe webhook signing secret (`STRIPE_WEBHOOK_SECRET`)
- Stripe products/prices configured in the account
- sandbox Worker URL and webhook endpoint configuration
- Playwright evidence against deployed sandbox Stripe test Checkout

## Automation Boundary

`pnpm smoke:stripe-sandbox` is the at-will automated Playwright sandbox smoke runner. It stays out of CI, targets the deployed Pages + sandbox Worker path, drives Stripe-hosted Checkout including 3D Secure, polls sandbox D1 remotely, and writes ignored evidence under `.codex-artifacts/stripe-sandbox-smoke/<run-id>/`.

Manual work remains starting `stripe listen --forward-to <sandbox-worker>/api/stripe/webhooks` before the run and keeping the sandbox Worker `STRIPE_WEBHOOK_SECRET` matched to that listener. The runner must not read or print Stripe secret values.
