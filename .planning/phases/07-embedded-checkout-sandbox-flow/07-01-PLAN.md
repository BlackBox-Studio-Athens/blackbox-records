# 07-01 Plan - Implement Worker APIs For Item Lookup, Offer Lookup, Trusted Session-Status Retrieval, And Checkout-Session Creation

## Objective

Implement the Worker-owned API surface required by the frontend checkout flow.

## Tasks

1. Implement item lookup, variant lookup, and ReadCheckoutState APIs.
2. Implement Checkout Session creation in the Worker backend.
3. Keep frontend contracts coupled to app identities and slugs, not raw Stripe IDs.

## Implementation Notes

- Public Worker API routes now exist for:
  - `GET /api/store/items/:storeItemSlug`
  - `GET /api/store/items/:storeItemSlug/variants`
  - `POST /api/checkout/sessions`
  - `GET /api/checkout/sessions/:checkoutSessionId/state`
- `StartCheckout` validates store item mapping, variant ownership, `ItemAvailability`, `OnlineStock`, and Stripe price mapping before creating a Checkout Session.
- Stripe SDK `22.1.0` targets API `2026-04-22.dahlia`, where embedded Checkout uses `ui_mode: embedded_page`.
- No frontend mount, webhook authority, order rows, stock decrement, reservation, shipping, or BOX NOW logic is part of this plan.

## Verification

- [x] ReadCheckoutState is Worker-owned
- [x] StartCheckout is Worker-owned
- [x] frontend contracts do not depend on raw Stripe IDs
