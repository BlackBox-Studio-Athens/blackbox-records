# 07-08 Plan - Refactor PDP And Checkout Entry Actions Around Add To Cart And Checkout

## Objective

Move the store item page from direct `Buy Now` handoff toward a familiar add-to-cart flow while preserving the Worker-owned checkout boundary.

## Scope Boundary

This commit changes entry actions and copy. It does not rebuild the checkout page layout yet.

## Tasks

1. Replace primary eligible PDP action with `Add To Cart`.
2. Opening `Add To Cart` stores or replaces the single item and opens the cart sheet.
3. Do not keep `Buy Now` as the primary label. Keep a secondary fast path only if it still sets the same cart item and routes through the canonical checkout URL.
4. Keep unavailable items viewable but prevent cart/checkout actions.
5. Remove phase/debug copy from shopper-facing checkout entry language.
6. Keep PDP visual treatment consistent with the design spike: editorial product detail first, cart affordance second.
7. Add tests for Add To Cart behavior, unavailable behavior, cart replacement behavior, and canonical checkout links.

## Verification

- [x] eligible item can be added to cart from PDP.
- [x] unavailable item cannot be added to cart.
- [x] checkout entry route uses canonical item-option slug.
- [x] shopper-facing PDP does not use `Buy merch`, `Variant`, `StoreItem`, or `StartCheckout` copy.
- [x] no browser payload includes Stripe price IDs or secrets.
- [x] Browser Use manual check verified the PDP Add To Cart flow, cart drawer contents, canonical checkout link, and unavailable-item non-actionable state.
- [x] `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No checkout page redesign yet.
- No return/retry UI.
- No order creation.
- No shipping forms.
