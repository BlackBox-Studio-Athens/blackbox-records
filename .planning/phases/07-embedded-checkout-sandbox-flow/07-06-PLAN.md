# 07-06 Plan - Add The Storefront Cart Icon And Single-Item Cart State Seam

## Objective

Add the browser-safe cart affordance and state seam that later cart drawer and checkout UI can consume.

## Scope Boundary

This commit introduces a single-item cart shell only. It does not render the full cart drawer and does not support multiple line items or quantity changes.

## Tasks

1. Add a frontend cart state module for one active item option.
2. Store only browser-safe values: canonical item-option slug, variant id, title, artist/subtitle, option label, image display data, and price display.
3. Include availability label only as display text; do not store authoritative stock counts.
4. Use replace-on-add semantics: adding a new item replaces the previous item rather than creating a second line.
5. Add a quiet line-style header cart icon button with accessible label and count badge.
6. Keep cart state local to the browser; do not add backend cart persistence.
7. Add tests proving no Stripe price IDs, server secrets, D1 IDs, stock counts, or order state enter browser cart state.

## Verification

- cart icon renders in the storefront shell.
- empty cart count is hidden or zero.
- setting one cart item shows count `1`.
- replacing the item keeps the cart single-item.
- cart state contains canonical item-option slug, not legacy `barren-point`.
- removing the item returns the cart to empty.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No cart drawer layout yet.
- No quantity selector.
- No discount code.
- No customer account state.
- No Worker cart API.
