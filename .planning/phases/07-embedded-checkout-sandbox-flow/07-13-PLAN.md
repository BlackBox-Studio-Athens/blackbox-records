# 07-13 Plan - Generate Local Mock Commerce State For Every Current Store Item

## Objective

Make `pnpm dev:stack:stripe-mock` prepare D1 with local-only mock commerce state for every current store item.

## Scope Boundary

This commit is local-development support. Fake local stock and `price_mock_*` mappings must not become sandbox or production inventory evidence.

## Tasks

1. Add a local commerce readiness manifest or generator that derives current store items from the frontend catalog source of truth.
2. Seed `StoreItemOption`, `ItemAvailability`, and `Stock` rows for every current store item in local mock mode.
3. Seed `VariantStripeMapping` rows with committed `price_mock_*` values for every current local mock variant.
4. Use clearly fake development stock defaults: `available`, `canBuy=true`, and positive `quantity`/`onlineQuantity`.
5. Keep real Stripe test Price IDs in ignored developer-local files only.

## Verification

- `pnpm dev:stack:stripe-mock` prepares local D1 without real Stripe keys.
- every current store item has a local `StoreItemOption`, `ItemAvailability`, `Stock`, and mock `VariantStripeMapping`.
- fake local stock is documented as mock-only development data.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No real stock counts.
- No sandbox seed requirement.
- No Stripe test Price ID commits.
- No webhook, order, or shipping behavior.
