# 07-15 Plan - Run Local Mock Checkout UAT Across Representative Item Types

## Objective

Prove the no-network local mock checkout path works across representative current release and distro items.

## Scope Boundary

This is local mock UAT only. It does not prove real Stripe payment, webhook authority, real inventory, or sandbox deployment readiness.

## Tasks

1. Run `pnpm dev:stack:stripe-mock` from a clean local state.
2. Exercise at least one release-derived item and multiple distro-derived items through PDP, cart, checkout, `StartCheckout`, and the frontend mock checkout panel.
3. Verify the UI is clear that mock checkout is local development behavior.
4. Capture any remaining item-specific content, slug, image, or summary issues as follow-up tasks.
5. Update docs with the representative local mock UAT path.

## Verification

- representative release item starts local mock checkout.
- representative distro item starts local mock checkout.
- readiness command passes before browser UAT.
- unavailable/unknown backend states remain understandable when local D1 is not prepared.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No real Stripe sandbox payment evidence.
- No webhook/order/stock decrement evidence.
- No production cutover.
- No BOX NOW shipping.
