# 07-07 Plan - Build The Shopify-Inspired Cart Drawer And Item Summary

## Objective

Build a familiar cart drawer using BlackBox styling and shadcn primitives, inspired by Shopify/Dawn cart patterns but not copied from Shopify theme code.

## Scope Boundary

The drawer presents one active item and routes to checkout. It does not become a true cart service.

## Tasks

1. Add a cart drawer component using shadcn `Sheet`, following `07-CART-CHECKOUT-DESIGN-SPIKE.md`.
2. Render empty and filled states.
3. Filled state shows image, title, artist/subtitle, option label, price, availability, subtotal, remove action, continue-shopping action, and checkout action.
4. Desktop sheet opens from the right and targets a narrow ecommerce side-panel width around `420-480px`.
5. Mobile sheet is full-height by default.
6. Opening the cart icon should open the drawer and return focus when closed.
7. Add tests for drawer state, accessibility labels, remove behavior, and checkout route generation.

## Verification

- cart icon opens the drawer.
- empty state is clear and offers continue shopping.
- filled state shows exactly one line item.
- remove clears the cart.
- checkout CTA points to the canonical item-option checkout URL.
- cart drawer uses the design spike's empty and filled-state copy.
- keyboard navigation and focus handling are acceptable.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No multi-line cart.
- No quantity stepper.
- No cart notes.
- No upsell/cross-sell blocks.
- No backend persistence.
