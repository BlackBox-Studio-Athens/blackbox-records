# 07-05 Plan - Correct Shopper-Facing Item Option URLs And Legacy Slug Redirects

## Objective

Fix the current route identity drift before cart UX work continues: shoppers buying `Disintegration` by `Afterwise` as `Black Vinyl LP` should see a URL that names that item option, not `/store/barren-point/`.

## Scope Boundary

This is a route and mapping correction. It does not add cart UI, Stripe behavior, order state, stock changes, or shipping.

## Tasks

1. Define the canonical item-option slug rule for current single-option store items: `{release-title}-{option-label}` normalized to lowercase kebab case, starting with `disintegration-black-vinyl-lp`.
2. Change the seeded/local smoke item from legacy `barren-point` shopper URL to an item-option URL such as `/store/disintegration-black-vinyl-lp/`.
3. Preserve `/store/barren-point/` and `/store/barren-point/checkout/` as compatibility redirects or aliases to the canonical item-option route.
4. Align frontend store paths, checkout paths, D1 seed mappings, mock Stripe seed mappings, and tests around the new canonical slug.
5. Ensure UI copy clearly separates:
   - artist: `Afterwise`
   - album/release: `Disintegration`
   - item option: `Black Vinyl LP`
6. Update the local mock checkout smoke path in docs from `/store/barren-point/checkout/` to `/store/disintegration-black-vinyl-lp/checkout/`.

## Verification

- `/store/disintegration-black-vinyl-lp/` renders the item detail page.
- `/store/disintegration-black-vinyl-lp/checkout/` is the canonical local smoke checkout path.
- legacy `/store/barren-point/` redirects or resolves safely to the canonical route.
- Worker offer lookup and `StartCheckout` still work for the canonical slug.
- no browser URL exposes Stripe IDs, D1 IDs, or backend table names.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No cart icon or drawer yet.
- No multi-item cart.
- No backend order schema.
- No webhook handling.
- No shipping fields.
