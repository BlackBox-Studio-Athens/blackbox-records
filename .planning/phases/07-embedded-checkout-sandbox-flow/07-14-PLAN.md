# 07-14 Plan - Add All-Items Local Mock Checkout Readiness Checks

## Objective

Add a deterministic local verification command that reports whether every current store item can enter the local mock checkout path.

## Scope Boundary

This is a readiness check, not a payment test. It must not call real Stripe, mutate sandbox state, or infer real stock truth.

## Tasks

1. [x] Add a repo-owned command that compares current store items against local D1 mock readiness state.

3. [x] Confirm fake local stock is positive only in mock/development readiness data.
4. [x] Fail clearly when local D1 has not been prepared.
5. [x] Document the command as part of the local mock stack maintenance contract.

- No production or sandbox writes.
