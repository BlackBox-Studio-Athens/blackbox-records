# 07-14 Plan - Add All-Items Local Mock Checkout Readiness Checks

## Objective

Add a deterministic local verification command that reports whether every current store item can enter the local mock checkout path.

## Scope Boundary

This is a readiness check, not a payment test. It must not call real Stripe, mutate sandbox state, or infer real stock truth.

## Tasks

1. Add a repo-owned command that compares current store items against local D1 mock readiness state.
2. Report missing `StoreItemOption`, `ItemAvailability`, `Stock`, or mock `VariantStripeMapping` rows by slug and source kind.
3. Confirm fake local stock is positive only in mock/development readiness data.
4. Fail clearly when local D1 has not been prepared.
5. Document the command as part of the local mock stack maintenance contract.

## Verification

- readiness command passes after `pnpm dev:stack:stripe-mock` preparation.
- readiness command fails with clear output when a mock mapping or local stock row is missing.
- browser-facing code still receives only app IDs and checkout eligibility, not Stripe Price IDs or D1 internals.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No real Stripe test validation.
- No automatic Stripe Product or Price creation.
- No production or sandbox writes.
- No webhook/order/shipping behavior.
