# 07-02 Plan - Add The Frontend Public Checkout API Client Seam

## Objective

Add a small frontend API seam for the public Worker checkout routes generated in `07-01`.

## Scope Boundary

This is a frontend data-access commit only. Do not change checkout UI behavior, mount Stripe.js, add return pages, or call live Stripe browser APIs.

## Tasks

1. Add a web-local public checkout API helper that uses `@blackbox/api-client`.
2. Use same-origin calls by default and `PUBLIC_BACKEND_BASE_URL` for local split-port development.
3. Expose typed helper methods for store offer lookup, variant list lookup, `StartCheckout`, and `ReadCheckoutState`.
4. Add unit tests for base URL handling, request payloads, response mapping, and visible error objects.

## Verification

- frontend helper consumes generated public API types
- no backend runtime imports appear in the web app
- no Stripe.js or embedded Checkout mount is added yet
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass

## Explicit Exclusions

- No checkout shell UI wiring
- No Stripe.js dependency
- No return or retry page
- No backend or OpenAPI changes
