# Phase 6.1 Research

## Repo facts that matter

- Phase 5 is now runtime-only.
- Phase 6 owns the unified shop projection and fixture-backed offer adapter.
- The project already chose D1 + Prisma for local commerce-state evolution, but had not yet inserted a dedicated phase for it.

## Research conclusions

1. D1 + Prisma should arrive after the storefront contract exists, not before.
2. The first D1-backed work should preserve the Phase 6 UI contract.
3. Phase 6.1 should stop before Stripe checkout logic and before the full order-lifecycle work that belongs to Phase 8.

## Resulting plan shape

- local D1 bootstrap
- Prisma runtime and migration setup
- repository seam evolution from fixture-backed state toward D1-backed reads where useful
