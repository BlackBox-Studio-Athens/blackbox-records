# Phase 6.1: Local Commerce State Foundation - Context

**Gathered:** 2026-04-20
**Status:** Planning complete

<domain>
## Phase Boundary

Phase 6.1 introduces local D1 and Prisma behind the shop repository boundary after the native storefront contract already exists. It does not introduce Stripe checkout, webhook semantics, or the full order lifecycle schema from Phase 8.

</domain>

<decisions>
## Implementation Decisions

### Local database scope
- **D-01:** Phase 6.1 bootstraps local D1 and binding names that match the later sandbox/production intent.
- **D-02:** Prisma is used for runtime access on D1, and migrations remain SQL applied through Wrangler.
- **D-03:** The first D1-backed reads in this phase are limited to local commerce-state concerns needed behind the shop repository boundary.
- **D-04:** Full order lifecycle and authoritative paid-state semantics remain deferred to Phase 8.

### Repository boundary
- **D-05:** The shop projection shape created in Phase 6 must not change in this phase.
- **D-06:** The repository seam may begin moving availability or inventory-related reads from fixture-backed data to D1-backed data where useful.
- **D-07:** Price authority is still not D1's responsibility; Stripe remains the later authority for catalog pricing.

### Migration workflow
- **D-08:** Prisma schema may model the local D1 shape, but migration SQL is generated and applied through the D1/Wrangler workflow.
- **D-09:** Manual dashboard edits are not the source of truth for schema evolution.

### Phase shape
- **D-10:** Phase 6.1 expands into 4 plans:
  - `06.1-01` local D1 bootstrap and bindings
  - `06.1-02` Prisma runtime setup on D1
  - `06.1-03` migration workflow baseline
  - `06.1-04` repository boundary swap toward D1-backed reads where needed
- **D-11:** Phase 7 depends on Phase 6.1 so checkout work starts only after the local state foundation exists.

### the agent's Discretion
- exact local DB filename and seed mechanics
- exact repository interface naming
- whether the first D1-backed read in this phase is availability-only or a slightly broader local commerce-state shape, as long as price authority remains outside D1

</decisions>
