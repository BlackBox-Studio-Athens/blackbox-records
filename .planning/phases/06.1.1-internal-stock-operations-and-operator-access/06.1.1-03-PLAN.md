# 06.1.1-03 Plan - Design The Static Astro Internal Stock Operations UI

## Objective

Create the thin protected operator UI for searching variants, viewing stock, recording changes and counts, and reviewing recent history. Astro owns the UI at `/stock/`; the Worker owns only the protected JSON API under `/api/internal/*`.

## Tasks

1. Design the stock overview and variant detail state on the protected hostname.
2. Design the `StockChange` and `StockCount` forms around fast reconciliation.
3. Design the history and attribution display so operators can see who changed stock and when.
4. Use `/stock/?variantId=<variantId>` for detail state so static Astro does not need to prebuild arbitrary D1 variant ids.

## Verification

- operators can find the right variant quickly
- current stock and online-available stock are both visible
- write actions and audit history are visible in one flow
