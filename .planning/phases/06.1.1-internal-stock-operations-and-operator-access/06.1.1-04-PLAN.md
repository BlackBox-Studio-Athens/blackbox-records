# 06.1.1-04 Plan - Lock Spreadsheet Policy And Reconciliation Rules

## Objective

Define how spreadsheets, offline events, and stock reconciliation interact with the D1-led stock ledger without creating dual sources of truth.

## Tasks

1. Define spreadsheets as temporary capture/reporting tools only.
2. Define when staff should use `StockChange` versus `StockCount` after offline events.
3. Define how `OnlineStock` stays conservative when offline sales are reconciled later.

## Verification

- D1 remains the only source of truth
- spreadsheet role is explicit and limited
- online/offline stock handling rules are explicit

## Output

- `06.1.1-STOCK-RECONCILIATION.md` locks the spreadsheet boundary, `StockChange` vs `StockCount` usage, and conservative `OnlineStock` policy.
- Phase 6.1.1 is complete after this plan; Phase 7 checkout API work is the next implementation focus.
