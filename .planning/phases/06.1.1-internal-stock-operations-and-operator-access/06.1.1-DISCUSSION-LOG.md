# Phase 06.1.1 Discussion Log

## Locked Defaults

- Use Google-backed Cloudflare Access for internal stock operations.
- Keep public storefront and shopper checkout unauthenticated.
- Put the internal stock tool on a separate protected backend hostname, not on the public shopper hostname.
- Do not reuse Decap auth for runtime stock writes.
- Use `CatalogItem`, `Variant`, `StockBalance`, `StockChange`, and `StockCount` as the domain language.
- Record `actor_email` and `recorded_at` on every stock write.
- Treat D1 as the authoritative stock ledger.
- Treat spreadsheets as temporary capture/reporting only.
- Track a conservative `online_available` quantity separately from total `StockBalance`.

## Why This Phase Exists

Checkout should not start depending on live stock if only one technical operator can reconcile offline bazaar and event sales. This phase creates the shared staff workflow and auth boundary that makes later checkout work operationally credible.

## Downstream Impact

- Phase 7 now depends on this phase so checkout consumes a defined stock and operator model.
- Phase 8 can build webhook-driven stock mutation on top of the same `Variant` and stock-ledger language.
- Later operator tooling can extend this thin stock tool without redoing auth or spreadsheet boundaries.
