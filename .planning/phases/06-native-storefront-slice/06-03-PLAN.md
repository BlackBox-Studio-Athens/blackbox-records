# 06-03 Plan - Build A Fixture-Backed Catalog Adapter

## Objective

Provide temporary offer state for the native store without polluting editorial content or requiring D1/Stripe.

## Tasks

1. Define fixture-backed offer records keyed by canonical shop slug.
2. Keep price and availability in the adapter layer rather than the content collections.
3. Expose offer state through the same interface later phases will preserve.

## Verification

- fixture adapter is separate from editorial content
- offer state is keyed by canonical shop identity
- later phases can swap adapter implementations without changing the shop projection shape
