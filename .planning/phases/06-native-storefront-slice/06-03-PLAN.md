# 06-03 Plan - Add The Temporary `ItemAvailability` Adapter

## Objective

Provide temporary variant-state reads through the same contract the future Worker API will use.

## Tasks

1. Define the temporary `ItemAvailability` shape.
2. Implement a temporary adapter that can satisfy the storefront without Worker, D1, or Stripe.
3. Keep the adapter API stable enough that later backend implementation swaps in underneath it.

## Verification

- temporary variant contract exists
- storefront can consume it
- future backend API can replace it without UI contract changes
