# 06-01 Plan - Define The Shared Shop Projection Contract

## Objective

Define one UI-facing shop read model that can represent both release-derived and distro-derived items.

## Tasks

1. Define the canonical projection fields consumed by shop list, PDP, and checkout shell routes.
2. Split content-derived fields from offer-state fields.
3. Make the projection stable enough that later D1 and Stripe integrations do not force route or component rewrites.

## Verification

- projection contract exists
- release and distro source kinds are explicit
- UI-facing shape is independent from any single backend source
