# 06-07 Plan - Reuse Editorial Assets And Summaries Without Duplication

## Objective

Ensure the native shop uses existing release and distro media/content instead of introducing a second editorial asset system.

## Tasks

1. Reuse release cover art and summary content for release-derived shop entries.
2. Reuse distro imagery and summary content for distro-derived entries.
3. Keep shop-only state limited to the adapter layer, not duplicated content payloads.

## Verification

- shop cards and PDPs reuse existing editorial assets
- no duplicate shop-only summary/image content is required
- the asset reuse rule is explicit for later D1 and Stripe phases
