# 05-02 Plan - On-Demand Route Boundary And Prerender Contract

## Objective

Keep the site editorially static-first while establishing the rule for which routes may become on-demand later.

## Tasks

1. Preserve prerender-by-default behavior for brochure/editorial routes.
2. Define the contract for commerce routes and endpoints to opt into on-demand rendering.
3. Keep the route boundary explicit so later phases do not accidentally turn the whole site into SSR.

## Verification

- brochure routes remain prerendered by default
- commerce-specific routes can opt into on-demand rendering deliberately
- the runtime split is visible in config or route conventions
