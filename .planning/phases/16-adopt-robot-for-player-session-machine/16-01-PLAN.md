# 16-01 Plan - Replace Player Session Reducer With Robot

## Objective

Replace the hand-rolled player session reducer with a `robot3` state machine while keeping existing app-shell player behavior intact.

## Tasks

1. Add `robot3` to `@blackbox/web`.
   - Use the latest published version at implementation time.
   - Keep it out of backend packages.

2. Characterize the current reducer before editing.
   - Run the existing player-session machine tests.
   - Add missing tests if any invalid-event no-op or state export behavior is not covered.

3. Implement the Robot machine.
   - Prefer preserving `reducePlayerSessionMachine(state, event)` as an adapter API.
   - Model current events only: `session-opened`, `iframe-loaded`, `player-surface-interacted`, `dismiss-requested`, `reopen-requested`, `stop-requested`.
   - Map Robot state/context back to the current `PlayerSessionMachineState` shape.

4. Update call sites only if the adapter cannot stay clean.
   - Keep changes limited to player-shell/session files.
   - Do not touch provider warmup, routing, overlays, or checkout.

5. Record validation evidence in `16-VALIDATION.md`.

## Verification

- `pnpm --filter @blackbox/web exec vitest run src/components/app-shell/player-session-machine.test.ts`
- `pnpm test:app-shell`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`
- Browser Use only if rendered player UI/event wiring changes.

## Stop Conditions

- Stop if Robot requires weakening the existing type model.
- Stop if migration changes player UX semantics.
- Stop if the work expands into app-shell routing or backend order state.
