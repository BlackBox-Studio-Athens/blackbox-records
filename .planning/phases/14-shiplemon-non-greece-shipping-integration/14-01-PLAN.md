# Plan 14-01: Integrate Shiplemon non-Greece shipping

> Status: post-MVP and stale. Replan or revise this file against `14-CONTEXT.md` before execution. Phase 14 must not
> block checkout launch readiness unless explicitly reactivated.

## Goal

Add a Worker-owned Shiplemon shipping path for non-Greece checkout while keeping Greece on the Phase 9 manual BOX NOW
path and preserving Stripe/webhook/order authority.

## Must Haves

- Greece checkout still uses the existing BOX NOW/manual-address behavior.
- Non-Greece checkout cannot start without a valid Worker-owned `shippingQuoteId`.
- Shiplemon API keys, raw rate IDs, shipment IDs, label URLs, invoice URLs, and raw provider payloads never reach browser
  code.
- Shiplemon shipment creation happens only after a verified paid Stripe webhook.
- Missing package profiles, missing credentials, stale quotes, provider failure, no rates, address mismatch, and shipment
  creation failure all fail closed or route to fulfillment review.

<task id="14-01-01" type="tdd">
<title>Add shipping persistence and package profile authority</title>
<read_first>

- apps/backend/prisma/schema.prisma

- apps/backend/prisma/migrations/
</read_first>
<action>
Add Prisma/D1 models and repository seams for `ShippingPackageProfile`, `ShippingQuote`, and `ShiplemonShipment`.
`ShippingPackageProfile` is keyed by `variantId` and stores weight in grams plus length, width, and height in centimeters.
`ShippingQuote` stores an opaque ID, CartDraft fingerprint, destination country, destination postal code/city when
provided, Shiplemon rate ID, display carrier/service names, amount, currency, estimated delivery days, package input,
expiration timestamp, and selected/consumed state. `ShiplemonShipment` links to `CheckoutOrder` and stores internal
shipment ID, carrier/service display data, tracking reference, tracking URL, label URL, invoice URL, status, failure
reason, and timestamps. Do not add fallback dimensions.
</action>
<acceptance_criteria>
- `apps/backend/prisma/schema.prisma` contains `ShippingPackageProfile`, `ShippingQuote`, and `ShiplemonShipment`.
- D1 migration SQL exists under `apps/backend/prisma/migrations/` and creates the same tables and indexes.
- Repository SPI exposes package-profile lookup, quote create/read/consume, and shipment create/update/read operations.
- Unit tests prove a missing `ShippingPackageProfile` makes non-Greece quote creation unavailable.
</acceptance_criteria>
</task>
