# 11-02 Plan - Rework Artist Detail Pages

## Objective

Make artist detail pages story-first while preserving direct route, overlay route, and app-shell player behavior.

## Scope

- Render artist Markdown body as the primary profile story when present.
- Keep `bio` as short summary/card/meta copy.
- Add a quiet profile link row near the profile story.
- Add an optional YouTube Videos section below the main story/latest-release area.
- Keep previous releases derived from release-to-artist relationships.
- Do not move music-player ownership out of the app shell.

## Files

- `apps/web/src/components/detail/ArtistDetailContent.astro`
- `apps/web/src/pages/artists/[slug].astro`
- `apps/web/src/pages/app-shell-overlay/artists/[slug].astro`
- `apps/web/src/components/cards/ReleaseCard.astro`
- `apps/web/src/lib/catalog-data.ts`
- `apps/web/src/styles/global.css`

</mitigations>
</threat_model>
