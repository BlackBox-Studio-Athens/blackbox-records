# 11-01 Plan - Extend Editorial Content Models

## Objective

Add the content-model and data-helper foundation needed for Phase 11 without changing commerce authority or route identity.

## Scope

- Artist detail content may gain optional profile links, optional YouTube video references, and rich Markdown body rendering support.
- `bio` remains the short/card/meta summary.
- Distro may gain optional known release dates and the new display group vocabulary.
- Existing slugs, images, routes, storefront handoff, and store item identity remain unchanged.

## Files

- `apps/web/src/content.config.ts`
- `apps/web/src/content/artists/*.md`
- `apps/web/src/content/distro/*.json`
- `apps/web/src/lib/admin/decap-config.ts`
- `apps/web/src/lib/distro-data.ts`
- `apps/web/src/lib/catalog-data.ts`

## Threat Model

<threat_model>
  <assets>
    <asset>Static content collections</asset>
    <asset>Browser-safe storefront metadata</asset>
    <asset>Worker-owned checkout, stock, payment, and order authority</asset>
  </assets>
  <risks>
    <risk severity="medium">Content schema additions accidentally expose backend/provider identifiers or commerce authority fields to the browser.</risk>
    <risk severity="medium">Distro group migration breaks existing content collection validation or store item metadata tests.</risk>
    <risk severity="low">YouTube URL modeling accepts arbitrary embed markup instead of controlled YouTube references.</risk>
  </risks>
  <mitigations>
    <mitigation>Keep new artist and Distro fields editorial/display-only.</mitigation>
    <mitigation>Model videos as structured YouTube references, not raw iframe HTML.</mitigation>
    <mitigation>Update unit tests and content schema checks in the same commit as data-helper changes.</mitigation>
  </mitigations>
</threat_model>
