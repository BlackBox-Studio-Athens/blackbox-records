# 11-01 Plan - Extend Editorial Content Models

## Objective

Add the content fields needed for richer artist profiles and clearer distro grouping while preserving existing content compatibility.

## Tasks

1. Extend the artist collection schema with optional profile links and optional videos.
2. Support multi-paragraph artist biography rendering without breaking existing single-string bios.
3. Extend distro content with optional release date and physical format group metadata for 12-inch vinyl, 7-inch vinyl, and CDs.
4. Update representative content entries only where source data is known; leave unknown release dates blank rather than inventing dates.
5. Keep store item projection browser-safe and free of Stripe, stock, payment, order, and backend secret data.

## Verification

- Collection schemas accept existing content.
- Updated sample artist and distro entries validate.
- No generated frontend payload exposes backend-only commerce fields.
