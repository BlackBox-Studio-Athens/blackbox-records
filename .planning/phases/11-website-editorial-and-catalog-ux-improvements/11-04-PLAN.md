# 11-04 Plan - Improve Distro Catalog Grouping And Copy

## Objective

Apply the new Distro grouping, optional release-date display, and editorial copy polish to the Distro catalog without changing commerce linkage.

## Scope

- Distro page groups display in this order: `Vinyl 12-inch`, `Vinyl 7-inch`, `CDs`, `Clothes`, `Tapes`, `Other`.
- `Other` is fallback-only for uncategorized display cases.
- Release dates are small optional metadata and omitted when unknown.
- Editorial copy may be tightened, but item identity, slugs, images, routes, and commerce handoff remain unchanged.

## Files

- `apps/web/src/pages/distro/index.astro`
- `apps/web/src/components/cards/DistroCard.astro`
- `apps/web/src/lib/distro-data.ts`
- `apps/web/src/lib/catalog-data.ts`

## Threat Model

<threat_model>
<assets>
<asset>Distro item route and store handoff identity</asset>
<asset>Static catalog content</asset>
<asset>Browser-facing metadata</asset>
</assets>
<risks>
<risk severity="medium">Distro copy or grouping changes accidentally alter item slugs, store URLs, or commerce metadata relied on by checkout work.</risk>
<risk severity="medium">Date display creates false precision by inferring unknown dates from prose.</risk>
<risk severity="low">New group labels break card layout on narrow screens.</risk>
</risks>
<mitigations>
<mitigation>Change display grouping and copy only; preserve slugs and store handoff fields.</mitigation>
<mitigation>Show dates only from explicit optional metadata.</mitigation>
<mitigation>Validate Distro route with Browser Use on desktop and a narrow viewport.</mitigation>
</mitigations>
</threat_model>
