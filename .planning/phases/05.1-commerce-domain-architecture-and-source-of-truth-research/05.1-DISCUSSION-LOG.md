# Phase 5.1 Discussion Log

## Locked Defaults

- Use a dual-deploy architecture:
  - static Astro frontend on GitHub Pages
  - separate Cloudflare Worker backend
- Use `Variant per catalog item` as the primary sellable unit.
- Treat Stripe as the authority for sellable commerce data needed for checkout.
- Treat D1 as the authority for operational state plus internal mappings.
- Treat Astro content as the authority for editorial content only.

## Why This Phase Exists

The previous milestone wording drifted toward “Astro on Workers.” This phase exists to stop implementation from mixing editorial content, Stripe catalog data, and D1 operational state without a clear model.

## Downstream Impact

- Phase 6 consumes `CatalogItem` plus temporary `VariantSnapshot`.
- Phase 6.1 introduces D1 + Prisma behind backend repositories.
- Phase 7 implements Worker APIs and Stripe checkout against the frozen contracts from this phase.
