# 10-04 Plan - Run Security, OpenAPI, Browser, And No-Secret Release Audit

## Objective

Audit the completed sandbox implementation before milestone review.

## Scope Boundary

This is audit and cleanup only. Do not introduce new commerce capabilities.

## Tasks

1. Audit browser code for leaked server secrets, Stripe price IDs, and backend runtime imports.
2. Audit public/internal OpenAPI separation.
3. Run browser checks for store, checkout, return, stock UI, and shell navigation.
4. Fix small inconsistencies discovered by the audit and document residual risks.

## Verification

- no-secret and API-boundary audit passes
- browser smoke checks are captured
- generated API artifacts are current
- `pnpm generate:api`, `pnpm test:unit`, `pnpm check`, and `pnpm build` pass

## Explicit Exclusions

- No production rollout
- No live Stripe validation
- No new order-management UI
- No new shipping scope
