# 10-02 Plan - Verify Sandbox Deployment, Secrets, D1, And Stripe Mapping Readiness

## Objective

Prove what can be verified about the sandbox environment before end-to-end UAT, and capture exact readiness blockers where Stripe or BOX NOW access is unavailable.

## Scope Boundary

This is readiness verification. Do not change application behavior unless a small docs/config fix is required to make documented commands true.

## Tasks

1. Verify Worker sandbox deployment path and required Cloudflare credentials.
2. Verify sandbox D1 migrations and expected seed/mapping state.
3. Verify Stripe sandbox secrets and price mappings are present or explicitly blocked without committing secret values.
4. Verify BOX NOW portal readiness is present or explicitly blocked without committing credentials.
5. Capture readiness evidence and blockers.

## Verification

- sandbox deployment and D1 commands are documented and verified
- required Stripe mappings are known
- BOX NOW portal readiness is known
- no secrets are committed
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass if any tracked files change

## Explicit Exclusions

- No live-mode keys
- No production D1
- No checkout feature changes
- No webhook behavior changes
