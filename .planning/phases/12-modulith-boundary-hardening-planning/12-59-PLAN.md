# 12-59 UI Foundation Boundary Extraction

## Summary

Shrink `platform-shared` further by extracting shared frontend UI primitives and the `cn` class-name helper into an
explicit closed `ui-foundation` module. This keeps shadcn/Radix-style primitives available as a documented interface
without treating them as platform bootstrap or business logic.

## Scope

- Add `ui-foundation` to the module map and manifest as the owner of:
  - `apps/web/src/components/ui/**`;
  - `apps/web/src/lib/utils.ts`.
- Move frontend modules that consume UI primitives or `cn` to depend on `ui-foundation` explicitly.
- Remove UI primitive roots and entrypoints from `platform-shared`.
- Add manifest validation coverage that rejects putting frontend UI primitives back under `platform-shared`.
- Keep existing import paths and component APIs unchanged.

## Out Of Scope

- No shadcn component rewrites.
- No styling, visual, or DOM behavior changes.
- No physical file moves.
- No package extraction, Nx adoption, or formal submodule modeling.
