# 12-41 Plan - Shell Player Session Controller

## Outcome

Move shell-owned player session lifecycle coordination out of `AppShellRoot` into a tested `player-shell`
controller while keeping React state, refs, and rendering in the composition root.

## Scope

- Extract player provider application, active session retirement, modal close/minimize/stop/reopen, frame-host sync,
  interaction marking, provider-origin warmup, and close-button focus scheduling into
  `player-shell/shell-player-session-controller`.
- Keep `AppShellRoot` responsible for player state/ref ownership and shell render composition.
- Preserve shopper-visible shell routing, overlay, player, mobile navigation, StoreCart, checkout, Worker, D1, Stripe, BOX
  NOW, generated API output, `.planning/config.json`, and worktree behavior.
