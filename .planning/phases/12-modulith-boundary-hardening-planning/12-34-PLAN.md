# 12-34 Plan - Player Modal Open Request Boundary Hardening

## Outcome

Move player modal open decision-making out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize player modal open requests:
  - requests without providers are ignored
  - opening the same release reuses the active session
  - opening a different release requests active-session stop before the new provider is applied
  - cached provider selection is preferred when available
  - unavailable cached provider selection falls back to the default provider
- Update `AppShellRoot` to delegate modal-open policy while preserving shell routing, overlays, player behavior,
  StoreCart behavior, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree
  behavior.

## Verification

- `pnpm test:unit -- shell-player-modal-open-request`
- Serena symbol overview/reference check for `resolvePlayerModalOpenRequest`
- `pnpm audit:module-boundaries`
- `pnpm depcruise:boundaries`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Browser Use

If Browser Use is available, validate player open, same-release reopen, different-release open, provider switching,
minimize/reopen/stop, and console cleanliness. If Browser Use remains unavailable, record the blocker in the summary
rather than substituting an external browser.
