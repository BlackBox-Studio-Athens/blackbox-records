# 12-32 Plan - Shell Document Click Intent Boundary Hardening

## Outcome

Move shell document click target classification out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize shell document click intent classification:
  - non-element-like targets are ignored
  - mobile navigation triggers outrank lower-priority anchors
  - player modal, mini-player open, and mini-player stop controls are classified before content navigation
  - player triggers return trigger/card details only when providers exist
  - player triggers without providers block lower-priority click handling
  - scroll target intent carries anchor fallback
  - plain anchors fall back to content navigation handling
- Update `AppShellRoot` to delegate click target classification while preserving shell routing, overlays, player behavior,
  StoreCart behavior, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree
  behavior.

## Verification

- `pnpm test:unit -- shell-document-click-intent`
- `pnpm audit:module-boundaries`
- `pnpm depcruise:boundaries`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Browser Use

If Browser Use is available, validate mobile nav toggle, player controls, scroll target links, section navigation, overlay
links, and console cleanliness. If Browser Use remains unavailable, record the blocker in the summary rather than
substituting an external browser.
