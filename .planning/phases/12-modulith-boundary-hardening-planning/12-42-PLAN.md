# 12-42 Plan - Decap YAML Builder Extraction

## Outcome

Move generic Decap YAML field and collection building out of the mixed `cms-admin` config hotspot into a tested helper
while preserving generated admin config output.

## Scope

- Extract scalar field mapping, block/list field rendering, schema hidden-field rendering, and file/folder collection
  rendering into `apps/web/src/lib/admin/decap-yaml-builder.ts`.
- Add direct characterization tests for the extracted builder behavior.
- Keep route/auth/base-path decisions and content collection definitions in `decap-config.ts`.
- Preserve shopper-facing catalog, checkout, stock, order, Stripe, D1, BOX NOW, Worker behavior, generated API output,
  `.planning/config.json`, and worktree behavior.
