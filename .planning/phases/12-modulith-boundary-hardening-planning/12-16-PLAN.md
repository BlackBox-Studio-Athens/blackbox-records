# 12-16 Plan - Shell Hero Scroll Progress Boundary Hardening

## Outcome

Move homepage hero scroll-progress coordination out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize hero scroll progress behavior:
  - progress uses the larger viewport/hero fade distance
  - progress is clamped between zero and one
  - homepage route sets the CSS custom property on the next animation frame
  - non-homepage routes do not set hero progress
  - cleanup removes listeners, cancels queued work, and clears the shell-owned CSS property
- Update `AppShellRoot` to delegate hero scroll progress wiring while preserving routing, overlay, player, StoreCart,
  checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree behavior.
