# 12-29 Plan - Cached Shell Page Restoration Boundary Hardening

## Outcome

Move cached shell page restoration orchestration out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize cached shell page restoration:
  - missing cached snapshot returns without shell side effects
  - cached shell section snapshot runs the section transition, snapshot apply, scroll reset, enter transition, and loading cleanup
  - failed snapshot application resets the section transition
  - cached non-section snapshot restores without section transition work
- Update `AppShellRoot` to delegate cached restoration while preserving shell routing, overlays, player behavior, StoreCart
  behavior, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree behavior.

## Verification

- `pnpm test:unit -- shell-cached-page-restoration`
- `pnpm audit:module-boundaries`
- `pnpm depcruise:boundaries`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

continuity, player continuity, StoreCart drawer behavior, and console cleanliness. If Browser Use remains unavailable,
record the blocker in the summary rather than substituting an external browser.
