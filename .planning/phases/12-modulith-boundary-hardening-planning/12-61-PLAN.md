# 12-61 Backend Infrastructure Boundary Extraction

## Summary

Shrink and close `platform-shared` by extracting backend Prisma/D1 persistence adapters and Stripe SDK integration into
explicit closed infrastructure modules.

## Scope

- Add a closed `commerce-persistence` module for `apps/backend/src/infrastructure/persistence/prisma/**`.
- Add a closed `stripe-integration` module for `apps/backend/src/infrastructure/stripe/**`.
- Move public commerce HTTP, orders, and operator stock dependency declarations to use the new modules explicitly.
- Remove backend persistence and Stripe entrypoints from `platform-shared`.
- Mark `platform-shared` closed once only bootstrap, config, URL, environment, and cross-cutting HTTP helper surfaces remain.
- Add manifest validation coverage that rejects reopening `platform-shared` or moving persistence/Stripe code back into it.

## Non-Goals

- No runtime behavior changes.
- No Prisma schema, D1 migration, Stripe API, generated API output, checkout contract, stock contract, or Worker behavior
  change.
- No new compatibility facades or deep-import support.
- No Nx, submodule model, worktree, or custom script change.
