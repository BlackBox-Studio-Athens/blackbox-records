# 12-30 Plan - Shell Popstate Navigation Boundary Hardening

## Outcome

Move shell `popstate` routing decisions out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize shell `popstate` routing:
  - overlay history entries restore overlays before shell section handling
  - missing overlay background href falls back to the current href
  - shell section history entries route through shell section navigation
  - cached non-section shell pages route through cached page restoration
  - unhandled routes close overlay state without restoring focus
- Update `AppShellRoot` to delegate `popstate` routing decisions while preserving shell routing, overlays, player behavior,
  StoreCart behavior, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree
  behavior.

## Verification

- `pnpm test:unit -- shell-popstate-navigation`
- `pnpm audit:module-boundaries`
- `pnpm depcruise:boundaries`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Browser Use

If Browser Use is available, validate browser back/forward behavior across overlay, shell section, cached page, and
fallback paths plus console cleanliness. If Browser Use remains unavailable, record the blocker in the summary rather
than substituting an external browser.
