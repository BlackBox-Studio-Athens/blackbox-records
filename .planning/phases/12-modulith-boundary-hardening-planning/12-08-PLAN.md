# 12-08 Plan - Player Iframe Session Boundary Hardening

## Objective

Move player iframe DOM construction, cache reuse, active/inactive marking, pruning, and retirement out of
`AppShellRoot` and into the closed `player` module without changing shell-owned player behavior.

## Scope

- player iframe creation attributes and event callbacks
- player iframe cache reuse and pruning
- active/inactive iframe state marking
- active session iframe retirement
- player module canvas and boundary manifest entrypoint updates

## Boundaries

- Preserve Bandcamp and Tidal iframe attributes, loading state, focus interaction marking, and cache behavior.
- Keep player modal state, provider selection, active session state, and shell-owned user behavior in `AppShellRoot`.
- Do not change shopper-visible UI, public checkout contracts, Worker behavior, D1, Stripe, BOX NOW, generated API output,
  `.planning/config.json`, or `workflow.use_worktrees`.
- Do not add temporary compatibility facades or widen the `app-shell` ownership root.
