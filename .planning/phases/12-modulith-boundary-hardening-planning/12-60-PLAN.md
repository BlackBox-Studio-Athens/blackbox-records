# 12-60 Operator Auth Boundary Extraction

## Summary

Shrink `platform-shared` by extracting Cloudflare Access operator identity parsing into an explicit closed
`operator-auth` module. Internal stock and internal order routes can both depend on this module without creating a
module cycle between `operator-stock` and `orders`.

## Scope

- Add `operator-auth` to the module map and manifest as the owner of:
  - `apps/backend/src/interfaces/http/auth/**`.
- Move `orders` and `operator-stock` to depend on `operator-auth` explicitly.
- Remove operator auth roots and entrypoints from `platform-shared`.
- Add manifest validation coverage that rejects putting operator auth code back under `platform-shared`.
- Keep existing import paths and auth behavior unchanged.

## Out Of Scope

- No Cloudflare Access behavior changes.
- No route behavior, operator UI behavior, generated API output, or auth header contract changes.
- No physical file moves.
- No package extraction, Nx adoption, or formal submodule modeling.
