# 12-06 Plan - App-Shell Overlay Loader Boundary Hardening

## Objective

Move the overlay fragment fetch/cache seam out of `AppShellRoot` while preserving the existing detail overlay behavior,
history behavior, focus behavior, and fallback-to-document-navigation behavior.

## Scope

- overlay fragment fetch/cache/in-flight request orchestration
- overlay prefetch failure handling
- `AppShellRoot.tsx` overlay loader delegation
- app-shell module canvas updates for the extracted seam

## Boundaries

- Preserve detail overlay open/close behavior for releases, artists, and news.
- Keep overlay state, history writes, focus restoration, abort coordination, and fallback navigation in `AppShellRoot`.
- Do not change shopper-visible UI, public checkout contracts, Worker behavior, D1, Stripe, BOX NOW, generated API output,
  `.planning/config.json`, or `workflow.use_worktrees`.
- Do not add temporary compatibility facades or widen the `app-shell` ownership root.
