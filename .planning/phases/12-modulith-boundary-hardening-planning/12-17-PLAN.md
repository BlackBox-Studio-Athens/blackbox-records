# 12-17 Plan - Shell Overlay Focus Boundary Hardening

## Outcome

Move overlay focus scheduling out of `AppShellRoot` into an internal app-shell helper with direct characterization
coverage.

## Scope

- Characterize overlay focus behavior:
  - connected overlay triggers are focused after close
  - disconnected overlay triggers are not focused
  - trigger focus restore is scheduled for the next animation frame
  - loaded overlay content scrolls to the top and focuses the close button on the next animation frame
- Update `AppShellRoot` to delegate overlay focus scheduling while preserving overlay history, route loading, fallback
  navigation, player, StoreCart, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`,
  and worktree behavior.
