# 12-05 Plan - App-Shell Boundary Hardening

## Objective

Move `app-shell` closer to its temporary-open exit criteria by adding direct characterization coverage and extracting the
shell page loading/cache seam from `AppShellRoot` without changing rendered shell behavior.

## Scope

- `AppShellRoot.tsx` shell section page loading and cache orchestration
- shell section navigation helper characterization
- existing StoreCart bridge and player helper coverage as compatibility guards
- app-shell module canvas updates for the extracted seam

## Boundaries

- Preserve top-level section navigation, overlay continuity, scroll/focus reset, player session behavior, and StoreCart
  drawer bridge behavior.
- Do not change shopper-visible UI, public checkout contracts, Worker behavior, D1, Stripe, BOX NOW, generated API output,
  `.planning/config.json`, or `workflow.use_worktrees`.
- Do not add temporary compatibility facades or widen the `app-shell` ownership root.

## Tasks

<task id="12-05-01" title="Characterize app-shell helper behavior">
  <action>
    Add direct unit coverage for shell section source classification, navigable shell and overlay anchors, shell-managed
    history marking, shell page request de-duplication, shell page cache reuse, and speculative prefetch failure handling.
  </action>
  <acceptance_criteria>
    The touched app-shell helper tests pass before and after the extraction.
  </acceptance_criteria>
</task>

<task id="12-05-02" title="Extract the shell page loader seam">
  <action>
    Move shell page fetch, parse, cache, in-flight request de-duplication, and prefetch failure handling into a dedicated
    app-shell helper. Keep `AppShellRoot` responsible for orchestration, history decisions, transition timing, and
    applying snapshots to the current document.
  </action>
  <acceptance_criteria>
    `AppShellRoot` calls the loader for cache reads, fetches, and prefetches while preserving existing shell navigation
    behavior.
  </acceptance_criteria>
</task>

<task id="12-05-03" title="Validate boundary and browser behavior">

  </action>
  <acceptance_criteria>

    console-cleanliness behavior.
  </acceptance_criteria>
</task>
