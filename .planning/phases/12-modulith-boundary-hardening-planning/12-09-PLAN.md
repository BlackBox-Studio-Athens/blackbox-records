# 12-09 Plan - Shell Portal Target Boundary Hardening

## Outcome

Reduce `AppShellRoot` direct ownership of route-scoped portal reinjection by extracting the duplicated portal-target
synchronization behavior into an internal `app-shell` helper with characterization coverage.

## Scope

- Characterize the existing portal target behavior before relying on the extraction:
  - inactive route clears the portal target
  - active route queries immediately
  - active route queries again on the next animation frame
  - cleanup cancels the scheduled animation frame
- Move Artists roster filter and Services inquiry portal-target synchronization to the helper.
- Keep the rendered portal components, shell routing behavior, overlay behavior, player behavior, StoreCart behavior, and
  checkout contracts unchanged.

## Out Of Scope

- No UI redesign.
- No changes to Worker behavior, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, GSD
  parallelization, or worktree settings.
- No new public app-shell entrypoint; this is an internal helper under the existing `app-shell` root.
