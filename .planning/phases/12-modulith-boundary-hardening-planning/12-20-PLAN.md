# 12-20 Plan - Player Shell View State Boundary Hardening

## Outcome

Move player shell view-state derivation out of `AppShellRoot` into a focused player helper with direct characterization
coverage.

## Scope

- Characterize player shell view-state behavior:
  - no active session returns the idle shell view state
  - active loading sessions keep the mini player hidden and use close copy
  - loaded, interacted, minimized sessions expose mini-player-ready state and provider labels
- Update `AppShellRoot` to delegate player UI state derivation while preserving session mutation, iframe behavior, modal
  controls, StoreCart, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree
  behavior.
