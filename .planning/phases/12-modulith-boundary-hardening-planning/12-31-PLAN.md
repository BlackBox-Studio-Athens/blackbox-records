# 12-31 Plan - Shell Escape Dismissal Boundary Hardening

## Outcome

Move shell Escape-key dismissal priority out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize Escape-key dismissal:
  - non-Escape keys are ignored without side effects
  - player modal dismissal takes priority over overlay dismissal
  - overlay history dismissal runs when no player modal is open
  - Escape is ignored when there is no dismissible shell state
- Update `AppShellRoot` to delegate Escape-key dismissal while preserving shell routing, overlays, player behavior,
  StoreCart behavior, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree
  behavior.

## Verification

- `pnpm test:unit -- shell-escape-dismissal`
- `pnpm audit:module-boundaries`
- `pnpm depcruise:boundaries`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Browser Use

If Browser Use is available, validate Escape behavior for player modal, overlay, and idle shell state plus console
cleanliness. If Browser Use remains unavailable, record the blocker in the summary rather than substituting an external
browser.
