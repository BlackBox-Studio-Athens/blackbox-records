# 12-27 Plan - Player Frame Host Boundary Hardening

## Outcome

Move player frame-host synchronization out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize player frame-host synchronization behavior:
  - missing active session or frame host is ignored
  - detached active iframe is appended, marked active, and reflected in player UI
  - already-attached active iframe is not reappended
- Update `AppShellRoot` to delegate frame-host synchronization while preserving player session mutation, iframe behavior,
  modal controls, StoreCart, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and
  worktree behavior.
