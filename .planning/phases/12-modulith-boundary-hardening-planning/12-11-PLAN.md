# 12-11 Plan - Route Loading Indicator Timer Boundary Hardening

## Outcome

Move route loading indicator timeout scheduling and cleanup out of `AppShellRoot` into an internal app-shell helper with
direct characterization coverage.

## Scope

- Characterize route loading timer behavior:
  - clearing an existing timer cancels it and resets the ref
  - clearing without a timer is a no-op
  - scheduling closes route loading after the default reset delay
  - scheduling replaces a pending timer
- Update `AppShellRoot` to delegate route loading timer mechanics while keeping route orchestration unchanged.
- Keep public routing, overlays, player behavior, StoreCart behavior, checkout contracts, Worker behavior, D1, Stripe,
  BOX NOW, generated API output, `.planning/config.json`, and worktree settings unchanged.
