# 12-07 Plan - App-Shell Overlay History Boundary Hardening

## Objective

Move overlay history write/collapse/back mechanics out of `AppShellRoot` while preserving existing detail overlay history,
focus restoration, and close behavior.

## Scope

- overlay history state shape
- overlay push/replace history writes
- overlay close-via-history-back behavior
- overlay collapse-to-background behavior
- `AppShellRoot.tsx` overlay history delegation

## Boundaries

- Preserve detail overlay URL behavior and browser back behavior.
- Keep overlay state ownership, close focus policy, and popstate branching in `AppShellRoot`.
- Do not change shopper-visible UI, public checkout contracts, Worker behavior, D1, Stripe, BOX NOW, generated API output,
  `.planning/config.json`, or `workflow.use_worktrees`.
- Do not add temporary compatibility facades or widen the `app-shell` ownership root.
