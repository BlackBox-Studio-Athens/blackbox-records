# 12-23 Plan - Player Session Machine State Boundary Hardening

## Outcome

Move player session-machine input derivation out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize player session-machine state derivation:
  - no active player session maps to the idle machine state
  - unloaded modal-open sessions preserve active session state
  - loaded, interacted, minimized sessions preserve active session state
- Update `AppShellRoot` to delegate session-machine input derivation while preserving player session mutation, iframe
  behavior, modal controls, StoreCart, checkout, Worker, D1, Stripe, BOX NOW, generated API output,
  `.planning/config.json`, and worktree behavior.
