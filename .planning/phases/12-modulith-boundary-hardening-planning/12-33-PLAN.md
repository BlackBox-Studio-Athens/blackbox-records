# 12-33 Plan - Shell Anchor Click Navigation Boundary Hardening

## Outcome

Move shell anchor-click navigation routing out of `AppShellRoot` into an internal app-shell helper with direct
characterization coverage.

## Scope

- Characterize shell anchor-click navigation:
  - cross-origin anchors are ignored
  - validated mobile navigation anchors close the mobile nav drawer
  - shell section anchors route through same-document shell navigation
  - overlay anchors set the overlay trigger and route through overlay loading
  - regular document navigation from an overlay collapses overlay history first
  - `data-astro-reload` anchors keep normal browser navigation behavior
- Update `AppShellRoot` to delegate anchor-click navigation while preserving shell routing, overlays, player behavior,
  StoreCart behavior, checkout, Worker, D1, Stripe, BOX NOW, generated API output, `.planning/config.json`, and worktree
  behavior.

## Verification

- `pnpm test:unit -- shell-anchor-click-navigation`
- Serena symbol overview/reference check for `routeShellAnchorClickNavigation`
- `pnpm audit:module-boundaries`
- `pnpm depcruise:boundaries`
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Browser Use

If Browser Use is available, validate header/footer/mobile section switching, overlay link opening, overlay-to-document
navigation, `data-astro-reload` behavior, and console cleanliness. If Browser Use remains unavailable, record the blocker
in the summary rather than substituting an external browser.
