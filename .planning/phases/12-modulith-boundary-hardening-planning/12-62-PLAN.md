# 12-62 App-Shell View Extraction

## Summary

Extract JSX-only shell render surfaces from `AppShellRoot.tsx` into a dedicated app-shell `view` folder. This keeps the
root focused on React composition, state, refs, and behavior wiring while making the app-shell folder structure easier to
scan.

## Scope

- Add `components/app-shell/view/` for render-only shell components.
- Extract:
  - mobile navigation sheet
  - overlay panel chrome
  - player modal and mini-player chrome
  - portal outlets for artist filters, services inquiry, and StoreCart button
- Keep routing, overlay history, player session coordination, StoreCart state mutation, and portal target discovery in
  `AppShellRoot.tsx`.
- Add render characterization tests for the extracted view surfaces.
- Update the app-shell module canvas and Phase 12 state docs.

## Non-Goals

- No visible UI copy, CSS class, route, overlay, player, StoreCart, or checkout behavior changes.
- No manifest ownership change; `components/app-shell/**` remains inside the closed `app-shell` module.
- No new visual design, copy, or interaction requirements beyond Browser Use verification that the extracted surfaces
  preserve the existing rendered shell behavior.
