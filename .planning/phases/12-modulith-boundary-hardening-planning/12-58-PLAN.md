# 12-58 Commerce Domain Boundary Extraction

## Summary

Shrink `platform-shared` by extracting backend commerce IDs and repository contracts into an explicit closed
`commerce-domain` module. This slice is an architecture-boundary refactor only: it must not change checkout, stock,
order, Prisma, HTTP, Stripe, D1, BOX NOW, generated API, or shopper-visible behavior.

## Scope

- Add `commerce-domain` to the module map and manifest as the owner of:
  - `apps/backend/src/domain/commerce/ids.ts`;
  - `apps/backend/src/domain/commerce/repositories/**`.
- Move checkout, orders, and stock backend modules to depend on the `commerce-domain` provided entrypoint instead of
  treating those contracts as `platform-shared`.
- Remove backend commerce domain roots and entrypoints from `platform-shared`.
- Add manifest validation coverage that rejects putting backend commerce domain code back under `platform-shared`.
- Keep existing import paths when they already target the domain repository barrel; do not add compatibility facades.

## Out Of Scope

- No physical file moves unless the manifest change exposes an actual boundary violation that requires one.
- No repository contract renames or behavior changes.
- No persistence, Prisma, D1, Stripe, webhook, checkout, stock, or order runtime changes.
- No generated API output changes.
- No Nx, submodule model, or package extraction.
