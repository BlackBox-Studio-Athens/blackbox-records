# 12-10 Plan - Shell Page Snapshot Application Boundary Hardening

## Outcome

Move shell page snapshot caching/application details out of `AppShellRoot` and into the existing `shell-page-snapshot`
seam, preserving same-document shell navigation behavior.

## Scope

- Characterize the existing snapshot application behavior:
  - readable document snapshots are cached through the shell page cache seam
  - missing shell main returns `null`/`false` without applying state
  - applied snapshots update main class/content, document metadata, rendered href callback, and active pathname callback
- Update `AppShellRoot` to delegate cache/apply DOM details while keeping route orchestration local.
- Keep routing, overlays, player behavior, StoreCart behavior, checkout contracts, Worker behavior, D1, Stripe, BOX NOW,
  generated API output, `.planning/config.json`, and worktree settings unchanged.
