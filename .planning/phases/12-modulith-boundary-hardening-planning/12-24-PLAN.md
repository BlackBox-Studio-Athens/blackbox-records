# 12-24 Plan - StoreCart Apply State Boundary Hardening

## Outcome

Move StoreCart state application and persistence out of `AppShellRoot` into the existing StoreCart bridge helper with
direct characterization coverage.

## Scope

- Characterize StoreCart state application behavior:
  - applied state is sent to the shell setter and persisted through configured storage
  - state is still applied when browser storage is unavailable
- Update `AppShellRoot` to delegate StoreCart state application while preserving StoreCart drawer behavior, checkout CTA
  paths, shell routing, overlays, player behavior, Worker, D1, Stripe, BOX NOW, generated API output,
  `.planning/config.json`, and worktree behavior.
