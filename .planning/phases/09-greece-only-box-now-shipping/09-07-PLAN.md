## Objective

Choose manual-address fulfillment as the Phase 9 implementation path and remove the locker-first requirement from new
checkout starts. New checkout sessions collect Greek shipping address/contact details inside Stripe Checkout, then
operators create BOX NOW shipments manually from paid order/payment tooling. The browser must not own BOX NOW locker
selection, shipping authority, BOX NOW credentials, voucher/label state, or fulfillment automation.

## Module Boundary

- `checkout-web` owns the shopper-facing shipping step copy and must send only app-owned checkout identities or CartLines
  to the public checkout API.
- `public-commerce-http` owns the public `StartCheckout` request contract and return-origin enforcement. It must not
  accept browser BOX NOW locker/address payloads for the manual path.
- `checkout-core` owns CartLine, quantity, stock, availability, Stripe mapping, and feature-gate validation. It creates
  pending orders with `shippingLocker: null` for the manual path.
- `stripe-integration` owns Stripe Checkout Session creation and enables Greece-only shipping address collection plus
  phone collection.
- `orders` and `commerce-persistence` continue to read legacy nullable locker snapshots for prototype/history rows, but
  this plan does not add a D1 address schema or rewrite historical pre-production migrations.
