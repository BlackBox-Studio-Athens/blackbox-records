# 15-01 Plan - Add Sharp-backed Asset QA

## Objective

Add a deterministic Sharp-backed asset QA command for committed web images without changing Astro rendering or public asset URLs.

## Tasks

1. Add `apps/web/scripts/check-assets.ts`.
   - Use `sharp().metadata()` for read-only inspection.
   - Resolve `apps/web/public` favicon/static assets and content-referenced local images.
   - Emit rule ids such as `favicon-size`, `favicon-alpha`, `artist-portrait-ratio`, and `unreadable-image`.
   - Print skipped unresolved classes explicitly instead of guessing.

2. Add tests for rule evaluation.
   - Cover passing metadata, wrong dimensions, bad aspect ratio, unreadable file, and stable diagnostic formatting.
   - Keep tests independent of the full content catalog when possible.

3. Wire scripts.
   - Add a web command such as `assets:check`.
   - Add a root wrapper only if it keeps the command discoverable without slowing `pnpm check` unexpectedly.

4. Update validation notes.
   - Create `15-VALIDATION.md` with command evidence and any intentionally skipped image classes.

## Verification

- `pnpm --filter @blackbox/web assets:check`
- Targeted asset QA tests.
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Stop Conditions

- Stop before rewriting image files.
- Stop before adding runtime image services or Worker image routes.
- Stop for human review if the check requires changing public assets or content image paths.
