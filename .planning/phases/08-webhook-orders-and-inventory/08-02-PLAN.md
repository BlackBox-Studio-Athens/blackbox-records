# 08-02 Plan - Add Order Repositories, Lifecycle Seams, And Typed Transition Guard

## Objective

Introduce backend repository, application seams, and the tiny typed transition guard for order lifecycle reads and writes.

## Scope Boundary

This commit creates internal seams only. Do not expose new public routes or handle Stripe webhooks yet.

## Tasks

1. Add order repository contracts in the commerce domain layer.
2. Add Prisma-backed order repository implementations.
3. Add application use cases for creating pending order state and reading order state.
4. Add a small backend order-state module with status union types, an allowed transition table, and a transition guard function.
5. Add tests for repository construction, pending/read behavior, valid transitions, invalid transitions, and duplicate paid replay no-op behavior.

## Verification

- repositories stay behind the persistence seam
- HTTP route files do not import Prisma
- use-case tests cover pending/read behavior
- transition tests prevent `paid -> not_paid`, old-order `not_paid -> pending_payment`, and browser/read-path persisted transitions
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass

## Explicit Exclusions

- No webhook route
- No Stripe signature verification
- No stock decrement
- No frontend UI
