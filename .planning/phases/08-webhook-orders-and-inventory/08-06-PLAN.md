# 08-06 Plan - Handle Unpaid, Expired, Canceled, And Needs-Review Outcomes

## Objective

Complete the non-paid order lifecycle outcomes without changing stock.

## Scope Boundary

This commit handles failed or uncertain outcomes only. It does not add refund, fulfillment, or operator order-management UI.

## Tasks

1. Map unpaid, expired, canceled, and ambiguous Stripe states to approved app-owned order states.
2. Ensure these paths never decrement stock.
3. Preserve traceability for needs-review cases.
4. Add tests for each non-paid outcome and duplicate webhook delivery.

## Verification

- unpaid and expired flows leave stock untouched
- needs-review states are persisted for later operator inspection
- checkout return state reflects app-owned order state where available
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass

## Explicit Exclusions

- No refund workflows
- No order admin UI
- No BOX NOW shipping
- No production incident tooling
