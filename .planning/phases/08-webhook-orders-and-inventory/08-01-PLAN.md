# 08-01 Plan - Add Minimal D1 Order Lifecycle Schema And Migration

## Objective

Add the minimum D1/Prisma schema needed to represent checkout order lifecycle state.

## Scope Boundary

This is schema-only. Do not add webhook routes, reconciliation logic, stock decrement, or frontend behavior.

## Tasks

1. Add Prisma models for the approved minimal order lifecycle.
2. Create an additive D1 migration through the established Wrangler migration workflow.
3. Keep Stripe identifiers backend-owned and mapped to order state without exposing browser authority.
4. Regenerate the committed Prisma client.

## Verification

- migration applies locally
- generated Prisma client compiles
- order status values match the approved vocabulary
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass

## Explicit Exclusions

- No webhook HTTP route
- No stock mutation
- No frontend reads
- No shipping fields beyond what is already approved for later phases
