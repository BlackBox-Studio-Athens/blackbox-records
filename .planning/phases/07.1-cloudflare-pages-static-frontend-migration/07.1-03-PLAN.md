# 07.1-03 Plan - Move Frontend Browser Env And Checkout Return-Origin Contracts To Cloudflare Pages

## Objective

Make Cloudflare Pages browser configuration and Worker checkout return-origin allowlists explicit for local, preview, sandbox, and production-branch deploys.

## Scope Boundary

Environment/config/docs only. Do not change checkout API shape, Stripe gateway behavior, webhooks, order state, or shipping.

## Tasks

1. Document required Cloudflare Pages browser env vars:
   - `PUBLIC_BACKEND_BASE_URL`
   - `PUBLIC_STRIPE_PUBLISHABLE_KEY`
2. Confirm `PUBLIC_CHECKOUT_CLIENT_MODE` is not a production Pages variable except for deliberate non-production testing.
3. Update Worker `CHECKOUT_RETURN_ORIGINS` docs for Cloudflare Pages preview and production origins.
4. Add no-wildcard origin rules for checkout returns.
5. Document local stack behavior remains unchanged.

## Verification

- Docs clearly separate browser-safe Pages env from Worker runtime secrets.
- Checkout return-origin docs include Cloudflare Pages origins and local origins only.
- No Stripe secret, webhook secret, D1 binding, or BOX NOW credential is listed as a Pages env var.
- `pnpm check` passes if config types change.

## Explicit Exclusions

- No live-mode Stripe keys.
- No broad wildcard return-origin allowlist.
- No frontend direct Stripe price IDs.
- No Worker route changes unless a typed config read is needed.
