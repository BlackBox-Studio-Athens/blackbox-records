# 07.1-04 Plan - Validate Cloudflare Pages Previews, Production Branch Deploys, And Worker API Routing

## Objective

Prove Cloudflare Pages can serve the static frontend and reach the Worker backend correctly before making it canonical.

## Scope Boundary

Validation and small fixes only. Do not remove GitHub Pages or start Phase 8 webhook/order implementation.

## Tasks

1. Trigger a Cloudflare Pages preview deployment from a non-production branch.
2. Trigger a production-branch Cloudflare Pages deployment.
3. Browser-test public storefront routes, checkout shell, checkout return/retry routes, and 404 behavior.
4. Verify frontend calls reach the intended Worker backend through `PUBLIC_BACKEND_BASE_URL`.
5. Verify checkout return URL creation accepts only the intended Pages/local origins.
6. Capture evidence and known gaps in the phase validation artifact.

## Verification

- Cloudflare Pages preview deploy succeeds.
- Cloudflare Pages production-branch deploy succeeds.
- Browser smoke passes for `/`, `/store/`, `/store/disintegration-black-vinyl-lp/`, `/store/disintegration-black-vinyl-lp/checkout/`, legacy `/store/barren-point/`, and `/404.html`.
- Checkout shell can call Worker offer/variant reads from the Pages origin.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass.

## Explicit Exclusions

- No webhook paid-order authority.
- No stock decrement.
- No shipping.
- No production DNS cutover without human approval.
