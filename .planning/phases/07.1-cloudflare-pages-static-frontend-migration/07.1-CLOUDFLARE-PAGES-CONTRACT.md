# Phase 7.1 Cloudflare Pages Deployment Contract

## Purpose

Cloudflare Pages becomes the future canonical static frontend host after Phase 7.1 acceptance. GitHub Pages remains the active deployment and rollback reference until that acceptance is complete.

## Static Artifact Boundary

- The frontend remains Astro `output: "static"`.
- The deploy artifact is the prebuilt `apps/web/dist` directory produced by `pnpm build`.
- Cloudflare Pages must not run backend code, Pages Functions, D1 access, Stripe secret access, webhooks, or operator API logic.
- The separate Cloudflare Worker remains the backend/BFF for `/api/*`, Stripe, D1, webhooks, stock operations, order state, and future BOX NOW work.

## CI And Project Contract

- Future Pages deployment should use GitHub Actions plus Cloudflare Pages Direct Upload.
- The CI job must run repo-owned gates before upload:
  - `pnpm test:unit`
  - `pnpm check`
  - `pnpm build`
- Required non-secret setup names:
  - Pages project: `blackbox-records-web`
  - production branch: `main`
  - deploy artifact: `apps/web/dist`
  - CI auth inputs: `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID`
- Do not commit Cloudflare account IDs, project IDs, tokens, custom domains, or account-specific values.

## Environment Boundary

Only browser-safe public variables belong in the Pages project:

- `PUBLIC_BACKEND_BASE_URL`
- `PUBLIC_STRIPE_PUBLISHABLE_KEY`
- `PUBLIC_CHECKOUT_CLIENT_MODE` only for non-production mock/dev contexts

Runtime business secrets stay out of Pages:

- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`
- D1 credentials/bindings
- Cloudflare Access policy secrets
- future BOX NOW credentials

## Rollback And Acceptance

- Keep `.github/workflows/pages.yml` and the GitHub Pages deployment active through Phase 7.1 acceptance.
- Do not treat Cloudflare Pages as canonical until preview and production-branch validation pass.
- Do not remove GitHub Pages or change canonical docs until Phase `07.1-05`.
- Checkout return-origin and CORS allowlists must be updated explicitly in later Phase 7.1 work; no wildcard origins.

## Explicit Non-Goals

- No workflow implementation in `07.1-01`.
- No DNS or custom-domain cutover.
- No Pages Functions.
- No Worker/backend consolidation into Pages.
- No production Stripe or checkout behavior change.
