# 07.1-02 Plan - Add Cloudflare Pages Project Config And Direct-Upload CI Workflow

## Objective

Add the first repo-owned Cloudflare Pages deployment path for the static Astro frontend using Wrangler Direct Upload from GitHub Actions.

## Scope Boundary

CI/config only. Do not change frontend routes, backend APIs, D1 schema, Stripe behavior, or remove GitHub Pages.

## Tasks

1. Add a GitHub Actions workflow for Cloudflare Pages static frontend deploy.
2. Reuse existing gates: `pnpm test:unit`, `pnpm check`, and `pnpm build`.
3. Upload `apps/web/dist` to the configured Cloudflare Pages project.
4. Add required CI secret/variable docs for `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`, and Pages project name.
5. Keep `.github/workflows/pages.yml` intact as rollback.

## Verification

- Workflow syntax is valid.
- Workflow does not expose runtime secrets to the browser.
- Build artifact path is `apps/web/dist`.
- `pnpm test:unit`, `pnpm check`, and `pnpm build` pass locally.

## Explicit Exclusions

- No DNS/custom-domain switch.
- No production Pages canonical docs yet.
- No Worker deploy changes unless required for origin allowlist documentation.
- No Pages Functions.
