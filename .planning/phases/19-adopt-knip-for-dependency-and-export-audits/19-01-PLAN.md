# 19-01 Plan - Add Knip Audit Configuration, Scripts, And Triage Documentation

## Objective

Add a scoped Knip audit command that reports unused dependencies, exports, and files across the monorepo without destabilizing public, generated, configured, or planning surfaces.

## Tasks

1. Add the dependency and command.
   - Add `knip@^6.14.2` or the latest compatible version to the root dev dependencies.
   - Add a root command such as `audit:unused` or `knip`.
   - Keep it outside `pnpm check` unless the first baseline proves quiet enough.

2. Configure workspace entrypoints and ignores.
   - Include package exports, scripts, Astro routes, backend entrypoints, generated API clients, GitHub workflows, and script entrypoints.
   - Exclude or classify generated Prisma/OpenAPI output and archived planning artifacts.
   - Keep ignore reasons specific, not broad.

3. Run and triage the first report.
   - Classify findings as real cleanup candidates, intentionally configured surfaces, generated surfaces, or deferred questions.
   - Do not delete route/content/generated/migration/export files unless usage evidence is clear.

4. Document the audit policy.
   - Update `.planning/codebase/STACK.md` or a nearby codebase doc with how Knip fits beside ESLint, dependency-cruiser, and custom audits.
   - Create `19-VALIDATION.md` with command output summary and any remaining known findings.

## Verification

- `pnpm audit:unused` or the chosen Knip command.
- `pnpm test:unit`
- `pnpm check`
- `pnpm build`

## Stop Conditions

- Stop before deleting any public route, content entry, generated file, D1 migration, package export, or workflow-referenced file based only on Knip output.
- Stop before replacing dependency-cruiser or existing module-boundary audits.
- Stop before making Knip mandatory in `pnpm check` if the first baseline is noisy.
