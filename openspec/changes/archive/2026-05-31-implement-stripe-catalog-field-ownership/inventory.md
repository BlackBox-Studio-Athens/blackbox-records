## Catalog Ownership Inventory

This inventory records the implementation baseline for the first slice of
`implement-stripe-catalog-field-ownership`. It is evidence for tasks 1.1-1.5.

## Store Item Projection Sources

- `apps/web/src/lib/catalog-data.ts` owns static Store Item projection from Astro
  content. Release Store Items are derived from `releases` entries through
  `createStoreItemFromRelease`, with special release slug overrides for
  `barren-point -> disintegration-black-vinyl-lp` and
  `caregivers -> caregivers-vinyl`. Distro Store Items are derived from `distro`
  JSON entries through `createStoreItemFromDistroEntry`.
- `apps/web/src/lib/store-page-data.ts` combines the Store Item projection with
  the primary availability adapter and deliberately avoids persisting or
  inventing browser-owned price authority.
- `apps/web/src/lib/item-availability.ts` has temporary static availability
  fixtures for `disintegration-black-vinyl-lp` and `afterglow-tape`; all other
  content Store Items fall back to unavailable/sold-out state.
- Content sources currently include release entries under
  `apps/web/src/content/releases/` and distro JSON entries under
  `apps/web/src/content/distro/`. Image fields are collection-owned assets, not
  stable public Stripe image URLs by themselves.

## D1 Rows And Seed Paths

- Schema ownership is in `apps/backend/prisma/schema.prisma`.
- `StoreItemOption`, `VariantStripeMapping`, and `ItemAvailability` are created
  in `apps/backend/prisma/migrations/0001_initial_commerce_state.sql`.
- `Stock`, `StockChange`, and `StockCount` are created in
  `apps/backend/prisma/migrations/0002_add_internal_stock_ledger.sql`.
- `StoreOfferSnapshot` is created in
  `apps/backend/prisma/migrations/0007_add_store_offer_snapshot.sql`.
- Local committed seed state in `apps/backend/prisma/seeds/local-commerce-state.sql`
  seeds `disintegration-black-vinyl-lp`, `aftermaths`, and `afterglow-tape`.
  Only `disintegration-black-vinyl-lp` is buyable there because it has available
  status and positive online stock.
- Ignored local Stripe test seed state in
  `apps/backend/prisma/seeds/local-stripe-test-state.sql` currently maps ten
  checkout-eligible variants to real test Price IDs. Provider IDs are
  intentionally not copied here.
- Local mock checkout state is generated by
  `apps/backend/scripts/seed-local-mock-commerce-state.ts`; it creates local-only
  fake stock, availability, `VariantStripeMapping`, and `StoreOfferSnapshot`
  rows for current Store Items.

## Stripe Catalog Sync Code

- `scripts/stripe-catalog-contract.ts` currently exposes one hardcoded catalog
  contract for `disintegration-black-vinyl-lp`.
- `scripts/stripe-catalog-verify.ts` reads D1 rows for variants from that
  hardcoded contract, builds in-memory repositories, runs `CatalogReconciler`,
  and prints a compact dry-run/apply report.
- `apps/backend/src/application/commerce/catalog-sync/` currently models Stripe
  Price authority, lookup key/metadata identity, redaction, Store Offer snapshot
  freshness, and fail-closed catalog drift. It does not yet model field-level
  ownership or Stripe Product projection drift separately.
- `apps/backend/src/infrastructure/stripe/stripe-catalog-gateway.ts` reads Prices
  with expanded Products, updates Price/Product metadata, archives Prices, and
  creates sandbox Prices. It does not yet expose a Product update operation for
  repo-owned Product Projection fields.

## Current Variant Classification

- First-slice checkout-eligible alignment targets are the D1 rows with available
  status and positive stock in sandbox/test seed state. The ignored local Stripe
  test seed currently contains ten such variants: `caregivers-vinyl`,
  `disintegration-black-vinyl-lp`, `afterglow-tape`,
  `crawl-eat-them-dead-or-alive-split-7`, `endless-searcher`,
  `in-your-absence`, `noise-without-decay`, `primal-ephemeral`, `selenopolis`,
  and `the-feathers-of-oblivion`.
- Committed local seed state treats `disintegration-black-vinyl-lp` as buyable,
  `aftermaths` as sold out with zero online stock, and `afterglow-tape` as sold
  out/future-buyable with physical stock but zero online stock.
- Content Store Items without D1 option, availability, stock, and mapping rows
  are report-only in this first slice. They must not trigger sandbox provider
  mutation until checkout eligibility exists in D1.

## Production Mutation Policy

- `stripe:catalog:verify` remains dry-run by default in every environment.
- `--apply` may mutate provider/D1 state only for `--env sandbox` in this
  change.
- Production catalog verification may report Product Projection, Price
  Authority, D1 readiness, Store Offer snapshot, and webhook readiness drift, but
  production apply remains rejected until the separate production go-live
  readiness workflow explicitly permits it.
